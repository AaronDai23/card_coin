## 问题一：Module 'app_settings' not found（Archive 失败）

### 现象

```
ios/Runner/GeneratedPluginRegistrant.m:18:9: error: Module 'app_settings' not found
** ARCHIVE FAILED **
```

### 根因

CI 执行 `pod install --repo-update` 时，`--repo-update` 触发 CocoaPods 刷新 spec repo 并重新解析依赖。  
`app_settings` Dart 包版本为 7.0.0，但其 iOS podspec 内写的是 `s.version = '6.1.2'`，版本不一致导致重新解析时该 pod 被静默丢弃。  
`flutter_install_all_ios_pods` 是唯一引用来源，没有兜底，因此 `app_settings` 不出现在 Pods 项目里，但 `GeneratedPluginRegistrant.m` 仍引用它，编译失败。

### 修复

**`ios/Podfile`**：在 `target 'Runner' do` 块中显式声明所有被 CI 静默丢弃的 Flutter plugin pod：

```ruby
pod 'app_settings',                :path => '.symlinks/plugins/app_settings/ios'
pod 'connectivity_plus',           :path => '.symlinks/plugins/connectivity_plus/ios'
pod 'image_picker_ios',            :path => '.symlinks/plugins/image_picker_ios/ios'
pod 'local_auth_darwin',           :path => '.symlinks/plugins/local_auth_darwin/darwin'
pod 'package_info_plus',           :path => '.symlinks/plugins/package_info_plus/ios'
pod 'pointer_interceptor_ios',     :path => '.symlinks/plugins/pointer_interceptor_ios/ios'
pod 'shared_preferences_foundation', :path => '.symlinks/plugins/shared_preferences_foundation/darwin'
pod 'sqflite_darwin',              :path => '.symlinks/plugins/sqflite_darwin/darwin'
pod 'url_launcher_ios',            :path => '.symlinks/plugins/url_launcher_ios/ios'
```

`flutter_install_all_ios_pods` 遇到重复声明会去重，本地构建不受影响。

**`.github/workflows/ios_build.yml`**：去掉 `pod install` 的 `--repo-update` 参数，防止重新解析覆盖已提交的 Podfile.lock：

```bash
# 修改前
pod install --clean-install --repo-update

# 修改后
pod install --clean-install
```

本地运行 `pod install` 更新 `Podfile.lock`（PODFILE CHECKSUM），与 Podfile 一同提交。

## 问题二：Cloud signing permission error（Export 失败）

### 现象

```
error: exportArchive Cloud signing permission error
error: exportArchive No profiles for 'com.cardbase.app' were found
** EXPORT FAILED **
```

### 根因

`ExportOptions.plist` 使用 `signingStyle = automatic`，在 Xcode 16+ 下 export 阶段会触发云端签名（Cloud-managed Distribution），要求 App Store Connect API Key 拥有 **Admin / Account Holder** 角色。  
当前 API Key 权限不足，且 `com.cardbase.app` 在 App Store Connect 无可用 Distribution Provisioning Profile。

### 修复

**`ios/ExportOptions.plist`**：

- `method`：`app-store` → `app-store-connect`（修复废弃警告）
- `signingStyle`：`automatic` → `manual`
- 新增 `provisioningProfiles` dict，由 CI 在运行时通过 PlistBuddy 填入 UUID

**`.github/workflows/ios_build.yml`**：新增步骤「Import provisioning profile」：

1. 解码 `IOS_PROVISIONING_PROFILE_BASE64` Secret 得到 `.mobileprovision`
2. 用 `security cms -D` + PlistBuddy 提取 UUID
3. 安装到 `~/Library/MobileDevice/Provisioning Profiles/`
4. 用 PlistBuddy 将 UUID 写入 `ExportOptions.plist:provisioningProfiles:com.cardbase.app`

`xcodebuild -exportArchive` 去掉 `-allowProvisioningUpdates` 和 `-authenticationKey*`（manual 签名不需要）。

**所需 GitHub Secret**：

| Secret | 内容 |
|--------|------|
| `IOS_PROVISIONING_PROFILE_BASE64` | App Store Distribution `.mobileprovision` 的 base64 编码 |

生成方式：
```bash
base64 -i comcardbaseappstore.mobileprovision | pbcopy
```

> **注意**：Provisioning Profile 中关联的证书必须与 `IOS_CERTIFICATE_BASE64` Secret 中的证书一致（均为 **Apple Distribution** 类型，`For use in Xcode 11 or later`）。

## 问题三：切换到 Xcode 26 打包

### 背景

macOS 15 runner 已预装 Xcode 26.3 / 26.2 / 26.1.1 / 26.0.1 及 iOS 26.x SDK，默认仍为 Xcode 16.4。无需换 runner 镜像。

### 修复

**`.github/workflows/ios_build.yml`**：checkout 后新增步骤：

```yaml
- name: Select Xcode 26.3
  run: |
    sudo xcode-select -s /Applications/Xcode_26.3.app
    xcodebuild -version
```

并在 `env` 中记录版本：

```yaml
env:
  XCODE_VERSION: '26.3'
```

`-sdk iphoneos` 在 Xcode 26 下自动解析为 `iphoneos26.x` SDK。
