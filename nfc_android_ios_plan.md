# 方案结论与落地规划（Android / iOS 双端）

## 一、整体结论

- **Android**：已得出可落地初步方案，可直接验证。  
- **iOS**：只能通过 **App Clip（轻应用）** 方式实现“碰 NFC → 唤起轻 App → 自动打开浏览器”，仍需进一步调研与配置验证。

## 二、Android 端方案（可执行）

### 1. 实现思路

- NFC 标签写入标准 **NDEF** 网页链接，拍卡时直接拉起系统或指定浏览器打开网页。

### 2. 关键实现

- 通过配置浏览器包名（如 Chrome：`com.android.chrome`）实现指定打开。  
- 拍卡时由默认浏览器按规则解析并打开 URI。

### 3. CAP 侧改造点

- CAP 需要支持 **AAR（Android Application Record）** 能力。  
- AAR 内容类似 NDEF 前缀，由客户端动态写入卡片。

### 4. 接口需求

- 在接口 `/m/smartCard/config` 新增一个返回字段，用于卡片写入配置。  
- 开发分工：核心开发方伟负责，我这边完成接入。

## 三、iOS 端方案（App Clip 轻应用）

### 1. 方案定位

- 零下载、访客友好，无需安装主 App。

### 2. 方案优势

- 用户无需下载完整 App。  
- 碰 NFC 后点击通知，秒开轻 App（≤15MB）。  
- 进入轻 App 后自动跳转浏览器打开目标页。

### 3. 配置要求

- NFC 标签写入 **App Clip 专属 HTTPS 唤起链接**（完整 URL）。  
- 该 URL 必须在 App Store Connect 配置为 App Clip 调用地址。  
- 配置 **Universal Link** 通用链接 + **AASA** 域名关联文件。  
- 轻 App 工程与主 App 域名绑定。  
- 推荐使用 **App Clip Code** 实体标签（识别优先级更高）。

### 4. 标准触发流程

1. 碰 NFC。  
2. 系统弹出轻 App 通知卡片。  
3. 用户点击打开。  
4. 瞬时拉起轻 App。  
5. 轻 App 启动后自动调用浏览器打开目标网页（无额外点击）。
